## Changes

M4(에이전트 스킬) 2차 사이클 — recap 파이프라인을 열어 M4를 완료했다. 구현
완료 후 git diff에서 변경의 "형태"를 도출하는 `/previs-recap` 스킬이 동작한다.
Grounding·Budget·시크릿 마스킹을 모델 재량에 맡기지 않도록, 구조화 블록
(file-tree·diff)은 신규 `@previs/recap` CLI가 git diff에서 기계 생성하고
서사·와이어프레임·리스크 판단만 스킬이 작성하는 하이브리드 구조다.

- **@previs/recap 패키지**: `@previs/schema` 선례를 따른 신규 워크스페이스
  패키지(private, ESM, plain tsc 빌드, Node 환경 vitest).
- **diff 소스 해석**: auto(base 감지 → 논리적 브랜치 동치 비교 →
  `merge-base..HEAD`, 더티 워킹트리는 오류 게이트) + `--range`/`--staged`/
  `--worktree` 오버라이드. worktree는 untracked 파일을 `ls-files --others`로
  수집하고 `--no-index` 합성 patch·numstat으로 포함(exit 0·1 성공 취급).
- **인벤토리**: `git diff --name-status -M -z`·`--numstat -z` 파싱(특수 경로·
  rename·바이너리 안전) → file-tree 블록 + churn·추가/삭제 산출.
- **발췌·Budget**: 파일별 unified diff를 hunk 단위로 분할해 완결 hunk 우선
  ~148줄 발췌(첫 hunk 초과 시 내부 절단), 절단 사실 note 기록. title은
  `<경로> (+추가 −삭제)` 70자 budget, churn 내림차순 랭킹, lockfile·생성물
  강등.
- **시크릿 마스킹**: 토큰 접두(sk-/ghp_/xox*/AKIA/JWT 등)·PEM 블록·`.env`
  값 전량·시크릿 계열 할당(따옴표 JSON 키·공백 포함 값 포함) 마스킹. CLI가
  출력하는 모든 diff 텍스트에 출력 전 일괄 적용.
- **manifest·CLI**: skip 판정(파일 1개 + ≤40줄), 치명 오류는 exit≠0,
  `errors` 비면 스킬이 발행 중단. 마스킹된 전체 patch는 out-of-line
  (`<manifest>.patches/`)에 저장하고 manifest엔 `fullDiffPath`만 기록(온디맨드
  로드). `--out` 필수. 루트 `pnpm recap:derive` 스크립트.
- **스키마**: `@previs/schema` `source.mode` optional enum(`range`/`staged`/
  `worktree`) 추가 — additive, 기존 문서·픽스처 무영향.
- **/previs-recap 스킬**: `skills/previs-recap/SKILL.md`(제품 디렉토리) +
  `.claude/skills/previs-recap` 심링크 도그푸딩. diff 소스 확정 → 도출 →
  skip·오류 게이트 → recap 표준 골격 조립(wireframe → prose → data-model/
  api-endpoint → file-tree → Key changes 탭 최대 8개) → doc:validate 검증
  루프. `disable-model-invocation: true`로 명시 호출 전용.
- **AGENTS.md**: 콘텐츠 규칙 3(Budget)에 후보 3개 미만이면 있는 만큼·0개면
  탭 생략이라는 상한 중심 해석 명시.
- **테스트**: recap 63건 추가 (총 245건). 도그푸딩 E2E: 실범위
  `v0.3.0..v0.4.1`(66파일) 도출 → 블록 조립 → doc:validate exit 0.
