## Changes

M5 로컬 런처. 세션마다 뷰어 dev 서버가 중복 기동되던 문제를 "재사용 우선"
싱글턴 런처로 해결한다. 스킬(`/previs-plan`·`/previs-recap`)이 발행 직후
`pnpm viewer:up`을 호출하면, 이미 떠 있는 뷰어가 있으면 그 URL을 반환하고
없으면 하나만 백그라운드로 기동한다. 다중 세션이 동시에 호출해도 프로세스는
정확히 하나만 살아남는다.

- **@previs/launcher (신규 패키지)**: 기존 CLI 패턴(plain `tsc` → `dist/` →
  `node`)을 따르는 얇은 오케스트레이터.
  - `paths.ts` — 포트·락·로그·docsDir 유도 단일 원천. `targetKey` =
    canonical docsDir(`PREVIS_DOCS_DIR` 오버라이드, 미존재 경로 안전 정규화)로
    포트·락·재사용 비교를 하나의 정체로 통일. `vite.config.ts`도 이 모듈을
    공유해 standalone·런처 포트가 일치한다.
  - `lock.ts` — `fs.open(…, 'wx')` 원자적 락 + `starting`/`running` phase
    판별 유니언. `running.pid`는 Vite 자신의 pid(정리 소유권 일치). stale
    판정은 phase별 pid 생존 확인, stale 복구는 별도 복구 락으로 TOCTOU 차단.
  - `port.ts` — 전용 대역(`47738~47801`) 내 가용 포트 탐색. 점유 포트는
    `/api/viewer-info`로 정체 확인해 우리 뷰어(docsDir 일치)면 채택(adopt),
    외부 프로세스면 +1. 락 없이 뜬 동일 뷰어의 중복 기동을 막는다.
  - `health.ts` — viewer-info 조회·재사용 판정.
  - `launch.ts` — 재사용 우선 흐름(락 재사용 → 채택 → 콜드스타트 spawn).
    detached 프로세스 그룹으로 기동하고, 실패 시 그룹 전체 SIGTERM/SIGKILL·
    생존 폴링 후에만 소유 락 제거(고아 Vite 방지). `cli/up.ts` 엔트리.
- **previs-runtime-plugin.ts (뷰어)**: dev 서버 프로세스 내부 수명주기 플러그인.
  `/api/viewer-info` 정체 응답, 모든 요청 활동 스탬프(`enforce: 'pre'`로 docs
  미들웨어보다 먼저), 유휴 30분 자동 종료, 종료 시 동기·소유권 확인 락 정리
  (SIGINT/SIGTERM 재-raise). `PREVIS_LOCK_PATH` 미설정(standalone)이면 비활성.
- **통합**: `vite.config.ts`가 `PREVIS_PORT`(런처 권위, `strictPort`)·런타임
  플러그인·공유 포트 유도를 반영. 루트 `pnpm viewer:up` 스크립트. 두 스킬의
  수동 `pnpm dev` 안내를 `pnpm viewer:up`으로 교체하고 `PREVIS_DOCS_DIR`
  오버라이드 안내 현행화.
- **테스트**: 36건 추가(총 281건). paths·lock·port·health 단위 + launch 통합
  (재사용·동시 재사용·락 없는 뷰어 채택·startup 실패 락 정리) + 런타임
  플러그인 미들웨어. 동시 콜드스타트 싱글턴·SIGTERM 락 정리는 E2E로 검증.
