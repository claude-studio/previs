## Changes

M4(에이전트 스킬) 1차 사이클 — previs의 "생산" 계층을 열었다. 에이전트가
계획을 JSON 블록 문서로 발행하고 뷰어가 수 초 내 로드하는 로컬 우선
파이프라인이 동작한다. `/previs-recap`(diff 기계 도출)은 2차 사이클(v0.5.0).

- **발행 서빙 API**: `apps/viewer/server/previs-docs-plugin.ts` —
  `GET /api/documents`가 `.previs/*.json`을 raw pass-through로 서빙
  (`PREVIS_DOCS_DIR` 오버라이드, 디렉토리 부재 시 빈 목록, 깨진 JSON은
  `errors`로 보고). 스키마 검증은 클라이언트 경계에 유지.
- **published origin**: `lib/documents.ts` `fetchPublishedDocuments()` —
  판별 유니언으로 전송 실패와 빈 디렉토리를 구분. `App.tsx`는 mount·window
  focus 시 갱신, 성공 응답에만 교체(실패 시 마지막 성공 목록 유지 +
  `publishedError` 경고, 성공 시 해제), id 충돌 우선순위
  `opened > published > builtin`, in-flight 가드.
- **origin 라벨**: `DocumentView`가 내장 픽스처/열린 파일/발행 문서 3종
  매핑으로 전환, `DocumentList`에 발행 새로고침 오류 배너 추가.
- **검증 CLI**: `packages/schema/src/cli/validate.ts` + 루트
  `pnpm doc:validate <file>` — safeParse 이슈 목록 출력, exit 코드 게이트.
- **/previs-plan 스킬**: `skills/previs-plan/SKILL.md` (제품 디렉토리) +
  `.claude/skills/previs-plan` 심링크 도그푸딩. plan 골격(헤드라인 →
  prose → file-tree → question-form → 승인 callout), `--wf-*`·시크릿
  마스킹 규칙, `plan-YYYYMMDD-<slug>` id·파일명 충돌 시 접미사 유일화,
  발행 후 doc:validate 검증 루프, 승인 게이트. `disable-model-invocation:
  true`로 명시 호출 전용.
- **스킬 명칭 확정**: `/plan`·`/recap` → `/previs-plan`·`/previs-recap`
  (내장 plan 모드와 충돌 회피) — AGENTS.md·ARCHI.md(§3·§10·§12)·GOAL.md
  일괄 갱신.
- **테스트**: 15건 추가 (총 177건 — viewer 126 + schema 51). 실서버 통합
  확인: 발행 → `/api/documents` 서빙 → doc:validate E2E.
